# 模型设置面板滚动修复报告

**修复日期**: 2026-03-19  
**问题**: 模型设置面板上下被固定，内容无法完全显示  
**状态**: ✅ 已修复

---

## 🐛 问题描述

### 现象

模型设置面板打开后，上下内容被固定住，无法滚动查看完整的模型列表。

### 影响

- 用户无法查看完整的模型列表
- 只能看到部分内容
- 体验极差

---

## 🔧 修复方案

### 问题原因

内容区域的容器虽然有 `overflow-y-auto`,但是没有明确的最大高度限制，导致滚动区域被父容器限制。

### 修复内容

**文件**: `src/app/components/ide/ModelSettings.tsx`

**修复前**:
```tsx
<div className="flex-1 overflow-y-auto min-h-0 p-5">
```

**修复后**:
```tsx
<div className="flex-1 overflow-y-auto min-h-0 p-5" style={{ maxHeight: "calc(88vh - 140px)" }}>
```

### 修复说明

- 添加明确的 `maxHeight` 样式
- 使用 `calc(88vh - 140px)` 计算可用高度
  - `88vh`: 面板总高度 (窗口高度的 88%)
  - `140px`: Header + Tabs 的高度
- 确保内容区域可以正常滚动
