# 隐藏 node_modules 文件指南

**问题**: IDE 或文件浏览器显示大量 `node_modules` 中的 TypeScript 类型定义文件

**影响**: 
- 文件列表混乱
- 搜索结果显示不必要的文件
- 性能下降

---

## ✅ 已修复

### VS Code 设置

已创建 `.vscode/settings.json` 文件，包含以下配置:

```json
{
  "files.exclude": {
    "**/node_modules": true,
    "**/dist": true,
    "**/build": true,
    "**/coverage": true
  },
  "search.exclude": {
    "**/node_modules": true,
    "**/dist": true
  },
  "files.watcherExclude": {
    "**/node_modules/**": true,
    "**/dist/**": true
  }
}
```

---

## 🔧 手动修复步骤

### VS Code

1. **打开设置**
   - macOS: `Cmd + ,`
   - Windows/Linux: `Ctrl + ,`

2. **搜索 "files.exclude"**

3. **添加模式**:
   - `**/node_modules`
   - `**/dist`
   - `**/build`
   - `**/coverage`

4. **搜索 "search.exclude"**

5. **添加相同模式**

### 重启 VS Code

```bash
# 完全重启
Cmd/Ctrl + Shift + P
输入 "Reload Window"
```

## 📋 其他 IDE 配置

### WebStorm/IntelliJ IDEA

1. 打开设置 (`Cmd/Ctrl + ,`)
2. 进入 **Project** > **Project Structure**
3. 右键点击 `node_modules` 文件夹
4. 选择 **Exclude**

### Sublime Text

创建 `Project.sublime-project`:

```json
{
  "folders": [
    {
      "path": ".",
      "folder_exclude_patterns": [
        "node_modules",
        "dist",
        "build",
        "coverage"
      ]
    }
  ]
}
```

### Atom

1. 打开设置 (`Cmd/Ctrl + ,`)
2. 进入 **Core**
3. 添加 **Ignored Names**:
   - `node_modules`
   - `dist`
   - `build`
   - `coverage`

## 🔍 故障排查

### 问题仍然存在？

1. **检查配置文件**
   ```bash
   cat .vscode/settings.json
   ```

2. **重置 VS Code 设置**
   - 删除 `.vscode/settings.json`
   - 重新创建文件

3. **检查 VS Code 扩展**
   - 禁用可能影响文件显示的扩展
   - 特别是文件浏览器相关扩展

### TypeScript 仍然显示错误？

1. **打开命令面板** (`Cmd/Ctrl + Shift + P`)
2. **输入**: `TypeScript: Restart TS Server`
3. **重启 VS Code**
