## 快速开始

### 系统要求

- **浏览器**: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- **网络**: 稳定的互联网连接（AI 功能需要）
- **存储**: 至少 5MB 可用空间（快照存储）

### 首次使用

1. **打开 IDE**: 启动 YYC3 Family AI 应用
2. **选择预览模式**: 根据需求选择实时/手动/延迟模式
3. **开始编码**: 在编辑器中编写代码，预览将自动更新
4. **创建快照**: 重要节点保存快照，方便回滚
5. **AI 辅助**: 使用 AI 功能生成、修复、优化代码

## 预览模式使用指南

### 三种预览模式对比

| 模式 | 更新策略 | 适用场景 | 性能影响 |
|------|---------|---------|---------|
| **实时模式** | 编辑后立即更新（100ms 节流） | 快速迭代、UI 调整 | 低 |
| **手动模式** | 手动触发更新 | 大文件编辑、复杂逻辑 | 最低 |
| **延迟模式** | 编辑后延迟 500ms 更新 | 平衡性能和体验 | 中 |

---

### 实时模式

**适用场景**:
- UI 样式快速调整
- 实时查看效果
- 小文件编辑

**使用方法**:

1. 点击预览模式切换按钮
2. 选择"实时模式"
3. 编辑代码，预览自动更新

**性能优化**:
- ✅ 100ms 节流控制
- ✅ 减少不必要的渲染
- ✅ 内存优化

**示例工作流**:

```
用户输入 → 编辑器监听 → 节流控制 (100ms) → 触发预览更新
```

**注意事项**:
- ⚠️ 大文件（> 500 行）建议使用延迟模式
- ⚠️ 复杂计算场景建议使用手动模式

---

### 手动模式

**适用场景**:
- 大文件编辑
- 复杂逻辑开发
- 需要精确控制更新时机

**使用方法**:

1. 点击预览模式切换按钮
2. 选择"手动模式"
3. 编辑代码，预览不会自动更新
4. 点击"刷新预览"按钮手动更新

**性能优势**:
- ✅ 完全控制更新时机
- ✅ 避免频繁渲染
- ✅ 最低性能开销

**示例工作流**:

```
用户输入 → 编辑器监听 → 标记待更新 → 用户点击刷新 → 触发预览更新
```

**提示**:
- 💡 编辑器会显示"有待处理更新"提示
- 💡 可使用快捷键 `Ctrl+Shift+R` (Windows/Linux) 或 `Cmd+Shift+R` (Mac) 刷新

---

### 延迟模式

**适用场景**:
- 平衡实时性和性能
- 中等复杂度项目
- 一般开发场景

**使用方法**:

1. 点击预览模式切换按钮
2. 选择"延迟模式"
3. 编辑代码，停止编辑 500ms 后自动更新

**性能优化**:
- ✅ 500ms 防抖控制
- ✅ 合并多次快速变更
- ✅ 智能延迟调整

**示例工作流**:

```
用户输入 → 编辑器监听 → 防抖控制 (500ms) → 合并变更 → 触发预览更新
```

**调整延迟时间**:

```typescript

// 可选范围: 300ms - 2000ms
```

## 快照管理使用指南

### 什么是快照？

快照是 IDE 在特定时刻保存的代码状态，包含：
- 所有打开的文件
- 文件内容和结构
- 元数据（时间、标签、描述）

### 快照容量限制

- **最大快照数**: 50 个
- **存储容量**: 5MB
- **自动清理**: 超过限制自动清理最旧快照

---

### 创建快照

**步骤**:

1. 点击"快照管理"按钮
2. 点击"创建快照"
3. 输入快照标签（必填）
4. 添加描述（可选）
5. 添加标签（可选）
6. 点击"保存"

**示例**:

```
标签: "v1.0 - 用户登录功能完成"
描述: "实现用户登录表单，包含验证和错误处理"
标签: ["feature", "auth", "milestone"]
```

**性能优化**:
- ✅ 自动压缩（> 10KB）
- ✅ 增量存储
- ✅ 快速创建（< 100ms）

---

### 恢复快照

**步骤**:

1. 打开"快照管理"
2. 选择要恢复的快照
3. 点击"恢复"按钮
4. 确认恢复操作

**注意事项**:
- ⚠️ 恢复会覆盖当前代码
- ⚠️ 建议先创建当前状态快照
- ⚠️ 恢复后可撤销（Ctrl/Cmd + Z）

**示例场景**:

```
场景: 功能开发出现严重问题，需要回滚

1. 打开快照管理
2. 选择"v1.0 - 功能正常"快照
3. 点击恢复
4. 代码回滚到正常状态
```

---

### 比较快照

**步骤**:

1. 打开"快照管理"
2. 选择两个快照
3. 点击"比较"按钮
4. 查看差异报告

**差异报告包含**:
- 新增文件
- 删除文件
- 修改文件
- 未变化文件

**示例**:

```
比较快照: v1.0 vs v1.1

新增文件: 2
  + src/utils/date.ts
  + src/utils/string.ts

删除文件: 1
  - src/deprecated.ts

修改文件: 3
  * src/App.tsx
  * src/index.tsx
  * src/styles.css

未变化: 10 文件
```

---

### 删除快照

**步骤**:

1. 打开"快照管理"
2. 选择要删除的快照
3. 点击"删除"按钮
4. 确认删除操作

**批量删除**:

1. 选择多个快照
2. 点击"批量删除"
3. 确认删除

**注意事项**:
- ⚠️ 删除操作不可撤销
- ⚠️ 建议定期清理不需要的快照

---

## 代码验证使用指南

### 什么是代码验证？

代码验证是 AI 生成代码后的质量检查，包含：
- 语法检查
- 安全性检查
- 最佳实践检查
- 代码复杂度分析

### 验证触发时机

代码验证会在以下场景自动触发：
1. AI 生成代码后
2. 用户应用代码前
3. 批量文件应用前

---

### 验证结果解读

#### 验证通过 ✅

```
✅ 验证通过
- 无语法错误
- 无安全隐患
- 符合最佳实践
- 复杂度: 低

操作: 可直接应用代码
```

#### 验证警告 ⚠️

```
⚠️ 验证警告
- 警告: 使用了 console.log，建议移除
- 建议: 考虑使用 TypeScript 类型注解

操作: 可应用代码，但建议优化
```

#### 验证失败 ❌

```
❌ 验证失败
- 错误: 检测到 eval() 使用，存在安全风险
- 错误: 文件过大（12000 行），超出限制

操作: 不可应用代码，需修复问题
```

---

### 验证指标

| 指标 | 说明 | 阈值 |
|------|------|------|
| **行数** | 代码总行数 | < 500 行 |
| **字符数** | 代码总字符数 | < 10000 字符 |
| **复杂度** | 代码复杂度评级 | 低/中/高 |

**复杂度评级**:
- **低**: 简单逻辑，易于理解
- **中**: 中等复杂度，需要一定理解
- **高**: 复杂逻辑，建议重构

---

### 手动触发验证

**步骤**:

1. 选择代码文件
2. 右键菜单 → "验证代码"
3. 查看验证报告

**批量验证**:

1. 选择多个文件
2. 右键菜单 → "批量验证"
3. 查看批量验证报告

---

### 验证最佳实践

#### 处理警告

```
建议优先级:
1. 修复安全性警告（高风险）
2. 修复最佳实践警告（中风险）
3. 修复代码风格警告（低风险）
```

#### 处理错误

```
必须修复的错误:
- 语法错误
- 安全漏洞
- 性能问题
- 文件过大
```

## AI 辅助功能使用指南

### AI 功能概览

| 功能 | 用途 | 触发方式 |
|------|------|---------|
| **生成代码** | 根据描述生成代码 | 自然语言描述 |
| **修复代码** | 修复错误和问题 | 描述问题 |
| **解释代码** | 解释代码逻辑 | 选中代码 + 提问 |
| **重构代码** | 优化代码结构 | 描述重构目标 |
| **生成测试** | 自动生成单元测试 | 选中代码 + 请求 |

---

### 使用 AI 生成代码

**步骤**:

1. 在 AI 输入框中描述需求
2. 点击"生成"按钮
3. 查看生成结果
4. 查看验证报告
5. 应用代码（如果验证通过）

**示例提示词**:

```
✅ 好的提示词:
- "创建一个用户登录表单，包含用户名、密码输入框和提交按钮"
- "为用户服务生成单元测试，覆盖登录、注册、密码重置功能"
- "修复登录页面的 404 错误，检查路由配置"

❌ 不好的提示词:
- "写个表单"
- "修bug"
- "测试"
```

---

### 意图识别

系统会自动识别用户意图：

| 用户输入 | 识别意图 | 系统行为 |
|---------|---------|---------|
| "创建一个登录表单" | `generate` | 使用生成模板 |
| "修复登录页面的报错" | `fix` | 使用修复模板 |
| "解释这段代码的作用" | `explain` | 使用解释模板 |
| "重构用户服务" | `refactor` | 使用重构模板 |
| "生成单元测试" | `test` | 使用测试模板 |

---

### Token 管理

**Token 估算**:

```
示例:
- "创建登录表单" → ~5 tokens
- "为用户服务生成包含登录、注册、密码重置功能的单元测试" → ~25 tokens
```

**上下文压缩**:

当上下文过长时，系统会自动压缩：
- 保留关键信息
- 移除冗余内容
- 压缩率 > 60%

---

### AI 最佳实践

#### 提供上下文

```
✅ 好的示例:
"在 React TypeScript 项目中，使用 Material-UI 创建一个用户登录表单，
包含用户名、密码输入框，支持表单验证和错误提示"

❌ 缺少上下文:
"创建登录表单"
```

#### 分步请求

```
对于复杂需求，建议分步请求:

步骤 1: "创建用户登录表单组件"
步骤 2: "添加表单验证逻辑"
步骤 3: "集成到现有页面"
```

## 常见问题解答

### 预览模式

#### Q: 为什么预览不更新？

**可能原因**:
1. 使用了手动模式，需要手动触发
2. 网络问题，预览服务未响应
3. 代码有语法错误

**解决方法**:
1. 检查预览模式设置
2. 刷新页面
3. 查看控制台错误

---

#### Q: 实时模式卡顿怎么办？

**可能原因**:
1. 文件过大（> 500 行）
2. 复杂计算逻辑
3. 频繁编辑

**解决方法**:
1. 切换到延迟模式
2. 或切换到手动模式
3. 减少同时打开的文件数

---

### 快照管理

#### Q: 快照保存失败怎么办？

**可能原因**:
1. 存储空间已满（5MB 限制）
2. 快照数量已达上限（50 个）
3. 浏览器隐私模式

**解决方法**:
1. 删除不需要的快照
2. 使用普通浏览器模式
3. 检查 localStorage 设置

---

#### Q: 如何恢复误删的快照？

**答**: 快照删除后无法恢复，建议：
1. 定期备份重要快照
2. 删除前仔细确认
3. 使用版本控制系统（Git）

---

### 代码验证

#### Q: 验证超时怎么办？

**可能原因**:
1. 文件过大
2. 网络问题
3. 服务器负载高

**解决方法**:
1. 分批验证大文件
2. 检查网络连接
3. 稍后重试

---

#### Q: 如何忽略某些验证规则？

**答**: 目前不支持自定义验证规则，建议：
1. 在代码注释中说明原因
2. 使用 `// @ts-ignore` 忽略类型检查
3. 联系团队添加自定义规则

---

### AI 功能

#### Q: AI 生成代码质量不高怎么办？

**解决方法**:
1. 提供更详细的需求描述
2. 提供项目上下文
3. 使用示例代码作为参考
4. 分步生成复杂功能

---

#### Q: AI 响应很慢怎么办？

**可能原因**:
1. 网络延迟
2. 提示词过长
3. 服务器负载高

**解决方法**:
1. 检查网络连接
2. 简化提示词
3. 稍后重试

---

#### Q: Token 估算不准确怎么办？

**答**: Token 估算是近似值，实际使用可能有差异：
- 英文估算较准确
- 中文估算可能偏高
- 建议实际使用测试
