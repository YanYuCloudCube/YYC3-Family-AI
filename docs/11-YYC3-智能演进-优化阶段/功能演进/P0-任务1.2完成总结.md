## 🎯 技术亮点

### 1. 哈希算法

使用简单的哈希算法快速比较文件内容：

```typescript
private calculateHash(content: string): string {
  let hash = 0;
  for (let i = 0; i < content.length; i++) {
    const char = content.charCodeAt(i);
    hash = ((hash << 5) - hash) + char;
    hash = hash & hash; // Convert to 32bit integer
  }
  return hash.toString(36);
}
```

**优点**:
- ✅ 计算速度快
- ✅ 无需第三方依赖
- ✅ 足够用于内容比较

## 💡 设计决策

### 1. 为什么选择Map而不是Object？

```typescript
// ✅ 更好
private snapshots: Map<string, Snapshot> = new Map();

// ❌ 不推荐
private snapshots: Record<string, Snapshot> = {};
```

**原因**:
- Map有明确的size属性
- Map的键可以是任意类型
- Map的遍历顺序是插入顺序
- Map有更好的性能（频繁增删）

### 2. 为什么限制50个快照？

```typescript
private readonly MAX_SNAPSHOTS = 50;
```

**考虑因素**:
- ✅ localStorage容量限制（约5MB）
- ✅ 用户体验（列表不会太长）
- ✅ 性能考虑（加载和比较速度）

### 3. 为什么使用简单哈希而不是MD5？

**原因**:
- ✅ 无需第三方依赖
- ✅ 计算速度快
- ✅ 只需要相等比较，不需要加密强度
- ✅ 纯原生实现
