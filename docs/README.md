### 文档更新原则
1. **及时性** - 功能变更后及时更新文档
2. **准确性** - 确保文档内容与代码一致
3. **完整性** - 提供完整的使用说明和示例
4. **可读性** - 使用清晰的语言和结构

### 文档命名规范
- 使用 `YYC3-[阶段]-[模块]-[主题].md` 格式
- 阶段：P0/P1/P2/P3/P5/P6
- 模块：架构/功能/优化/测试等
- 主题：具体功能或文档类型
