# YYC³ CloudPivot Intelli-Matrix - 文档命名规范

## 📋 命名规范

### 项目级文档 (`/docs/`)

#### 目录命名规范

**格式**: `{序号}-{阶段}-{功能}-{子功能}`

**示例**:
```
00-YYC3-项目总览-目录索引/
01-YYC3-项目规划-启动阶段/
02-YYC3-项目规划-设计阶段/
03-YYC3-项目开发-实施阶段/
04-YYC3-项目测试-审核阶段/
05-YYC3-项目部署-文档闭环/
06-YYC3-项目运营-维护阶段/
07-YYC3-项目合规-安全保障/
08-YYC3-项目整合-实施阶段/
09-YYC3-项目资产-知识管理/
10-YYC3-项目模版-标准规范/
11-YYC3-智能演进-优化阶段/
12-YYC3-用户指南-操作手册/
13-YYC3-智能演进-优化阶段/
```

**规则**:
- 序号: 2 位数字（00-99）
- 阶段: 项目规划、项目开发、项目测试、项目部署、项目运营、项目合规、项目整合、项目资产、项目模版、智能演进、用户指南
- 功能: 简短描述（中文）
- 子功能: 可选，进一步细分

#### 文件命名规范

**格式**: `{序号}-{项目}-{功能}-{描述}.md`

**示例**:
```
001-项目总览索引-项目总览手册.md
002-项目总览索引-快速开始指南.md
003-代码标头规范标准.md
004-项目多维检测审查报告.md
```

**规则**:
- 序号: 3 位数字（001-999）
- 项目: CP-IM（CloudPivot Intelli-Matrix）
- 功能: 简短描述（中文）
- 描述: 详细描述（中文）

#### 标准规范文档命名

**格式**: `YYC3-{功能}-{描述}.md`

**示例**:
```
YYC3-代码标头规范标准.md
YYC3-代码标头规范化最终总结报告.md
YYC3-项目多维检测审查报告.md
YYC3-文档架构整理报告.md
YYC3-封装文档更新指南.md
```

### 应用级文档 (`/src/app/docs/`)

#### 文件命名规范

**格式**: `{功能}-{类型}.ts`

**示例**:
```
API-REFERENCE.ts
COMPONENT-REFERENCE.ts
DEVELOPER-HANDOFF.ts
TESTING-GUIDE.ts
ALIGNMENT-PLAN.ts
FINAL-AUDIT-REPORT.ts
HA-ARCHITECTURE-TEST.ts
LOCAL-DESIGN-GUIDE.ts
STORAGE-AUDIT.ts
```

**规则**:
- 功能: 大写字母，单词用连字符分隔
- 类型: REFERENCE, GUIDE, HANDOFF, REPORT, TEST, PLAN, AUDIT

## 🎯 命名原则

### 1. 一致性

- 所有文档使用统一的命名格式
- 相同类型的文档使用相同的后缀
- 序号保持连续性

### 2. 可读性

- 使用中文描述，便于理解
- 使用连字符分隔，便于解析
- 避免使用特殊字符

### 3. 可扩展性

- 预留足够的序号空间
- 支持多层子目录
- 支持新类型的文档

### 4. 标准化

- 遵循行业最佳实践
- 符合 YYC³ 标准化要求
- 便于自动化处理
