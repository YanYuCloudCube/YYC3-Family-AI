# YYC³ AI 提示词系统架构总览

## 📋 系统概述

YYC³ AI 提示词系统是一个**基于技术性分类的阶段化提示词系统**，旨在形成**准确、完整、闭环**的提示词体系。系统按照**技术栈**和**开发阶段**进行双重分类，每个阶段都是**一次可实现的**，确保提示词的**可执行性**和**完整性**。

### 核心设计理念

1. **技术性分类**：按照技术栈进行分类，确保每个提示词专注于特定技术领域
2. **阶段化实现**：按照开发阶段进行分类，确保每个阶段都是一次可实现的
3. **闭环设计**：每个阶段都有明确的输入、输出和验收标准
4. **变量词整合**：所有变量词直接融入提示词，无需额外配置
5. **全维度覆盖**：覆盖架构、功能、性能、安全、测试、部署等全维度

## 🔄 闭环设计

### 每个阶段的闭环流程

```
┌─────────────────────────────────────────────────────────────┐
│                    阶段开始                              │
└─────────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│  1. 输入定义（Input Definition）                         │
│     - 前置条件（Prerequisites）                           │
│     - 依赖关系（Dependencies）                            │
│     - 输入数据（Input Data）                              │
└─────────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│  2. 提示词执行（Prompt Execution）                        │
│     - 技术栈（Tech Stack）                                │
│     - 实现细节（Implementation Details）                   │
│     - 代码示例（Code Examples）                            │
└─────────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│  3. 输出生成（Output Generation）                         │
│     - 生成代码（Generated Code）                           │
│     - 配置文件（Config Files）                             │
│     - 文档（Documentation）                               │
└─────────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│  4. 验收标准（Acceptance Criteria）                       │
│     - 功能测试（Functional Tests）                         │
│     - 性能测试（Performance Tests）                         │
│     - 代码质量（Code Quality）                             │
└─────────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│                    阶段完成                              │
│              ✅ 进入下一阶段                               │
└─────────────────────────────────────────────────────────────┘
```

### 闭环验证机制

每个阶段都包含以下验证机制：

1. **前置条件检查**：确保前置阶段已完成
2. **依赖关系验证**：确保所有依赖已满足
3. **输入数据验证**：确保输入数据格式正确
4. **输出结果验证**：确保输出符合预期
5. **验收标准检查**：确保满足所有验收标准

## 🎯 使用指南

### 1. 按阶段使用

**推荐路径**：按照 P0 → P1 → P2 → P3 的顺序依次执行

**优点**：

- 确保每个阶段都是一次可实现的
- 每个阶段完成后都有可运行的系统
- 便于逐步验证和调试

**步骤**：

1. 执行 P0-核心架构阶段的所有提示词
2. 验证 P0 阶段的验收标准
3. 执行 P1-核心功能阶段的所有提示词
4. 验证 P1 阶段的验收标准
5. 依次执行 P2 和 P3 阶段

### 2. 按技术使用

**推荐路径**：按照技术栈的依赖关系执行

**优点**：

- 专注于特定技术领域
- 便于技术栈学习和掌握
- 便于团队分工协作

**步骤**：

1. 执行前端框架相关的所有提示词
2. 执行状态管理相关的所有提示词
3. 执行布局引擎相关的所有提示词
4. 依次执行其他技术相关的提示词

### 3. 按需使用

**推荐路径**：根据项目需求选择性执行

**优点**：

- 灵活性高，适应不同项目需求
- 避免不必要的功能开发
- 快速实现核心功能

**步骤**：

1. 识别项目需要的技术和功能
2. 选择对应的提示词文档
3. 按照依赖关系执行提示词
4. 验证执行结果

## 🔧 变量词整合

### 变量词使用原则

1. **直接融入**：所有变量词直接融入提示词，无需额外配置
2. **统一格式**：使用 `{{VARIABLE_NAME}}` 格式表示变量
3. **默认值**：每个变量都有默认值，确保提示词可独立执行
4. **可覆盖**：用户可以通过替换变量值来定制提示词

### 变量词分类

| 分类 | 变量示例 | 说明 |
|------|----------|------|
| **项目信息** | `{{PROJECT_NAME}}`, `{{TEAM_NAME}}`, `{{CONTACT_EMAIL}}` | 项目基本信息 |
| **技术栈版本** | `{{REACT_VERSION}}`, `{{TYPESCRIPT_VERSION}}`, `{{VITE_VERSION}}` | 技术栈版本号 |
| **配置参数** | `{{PORT}}`, `{{API_TIMEOUT}}`, `{{CACHE_TTL}}` | 配置参数 |
| **品牌标识** | `{{BRAND_NAME}}`, `{{BRAND_SLOGAN}}`, `{{ICON_LIBRARY}}` | 品牌标识信息 |

### 变量词替换

在使用提示词前，可以替换以下变量：

```bash
# 项目信息
PROJECT_NAME="YYC3 AI Code"
TEAM_NAME="YanYuCloudCube Team"
CONTACT_EMAIL="admin@0379.email"

# 技术栈版本
REACT_VERSION="18.3.1"
TYPESCRIPT_VERSION="5.3.3"
VITE_VERSION="5.0.12"

# 配置参数
PORT="3201"
API_TIMEOUT="30000"
CACHE_TTL="3600"

# 品牌标识
BRAND_NAME="YYC³ Family AI"
```

#### 2. 递交技巧

**分块递交**：

- 如果提示词过长，可以分块递交

- 避免频繁切换对话
- 使用相同的会话继续执行

- 记录问题和解决方案
- 及时调整提示词

#### 3. 错误处理

**常见错误**：

| 错误类型 | 原因 | 解决方案 |
|----------|------|----------|

| 代码无法运行 | 环境配置问题 | 检查依赖和配置 |
| 功能不完整 | 提示词遗漏 | 补充提示词，重新递交 |
