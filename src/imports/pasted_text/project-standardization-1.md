## 任务目标

对整个项目进行标准化层面的全面检查和优化，确保所有模块、组件、文档和交互保持高度统一。

## 检查范围

1. **设计语言统一**
   - 颜色系统统一
   - 排版系统统一
   - 组件样式统一
   - 图标系统统一
   - 交互模式统一

2. **代码规范统一**
   - 命名规范统一
   - 代码格式统一
   - 注释风格统一
   - 文件结构统一
   - 导入导出方式统一

3. **文档风格统一**
   - API 文档格式统一
   - 组件文档格式统一
   - 用户文档格式统一
   - 开发文档格式统一
   - 注释模板统一

4. **交互体验统一**
   - 表单交互统一
   - 弹窗行为统一
   - 加载状态统一
   - 错误提示统一
   - 快捷键统一

## 执行步骤

1. 梳理现有设计语言规范
2. 检查所有组件和页面的设计一致性
3. 检查所有代码文件的规范一致性
4. 检查所有文档的格式一致性
5. 检查所有交互行为的一致性
6. 生成统一化报告和改进建议
7. 实施统一化改进

## 验收标准

✅ 设计语言完全统一
✅ 代码规范完全统一
✅ 文档风格完全统一
✅ 交互体验完全统一
✅ 统一化报告完整
✅ 改进建议全部实施

## 输出要求

1. 生成统一化检查报告，包括：
   - 发现的不一致问题
   - 统一化建议
   - 实施计划

2. 生成统一化规范文档，包括：
   - 设计语言规范
   - 代码规范
   - 文档规范
   - 交互规范

3. 生成改进后的代码和文档文件
