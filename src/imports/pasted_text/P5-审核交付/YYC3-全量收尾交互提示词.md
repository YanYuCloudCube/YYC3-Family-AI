# YYC³ Figma AI 全量收尾交互提示词系统

> @source: GitHub YYC-Cube/YanYuCloud/YYC3-Design-Prompt/P5-审核交付/
> @version: v1.0.0 | @status: stable

## 概述

本提示词系统用于指导 AI 完成项目的全量收尾工作，确保代码质量、功能完整性、测试覆盖率和系统稳定性。

## 12 大分类

| # | 分类 | 核心验收标准 |
|---|------|-------------|
| 1 | **代码语法类** | TS 编译零错误、ESLint 全通过、无 Console 警告、JSDoc > 90%、代码规范统一、无循环依赖/死代码 |
| 2 | **功能完整逻辑类** | 核心功能完整、业务逻辑正确、首屏 < 2s、UX 流畅、安全达标、跨平台兼容 |
| 3 | **测试用例类** | 单元 > 80%、集成 > 70%、E2E 覆盖主流程、性能/安全/兼容性测试通过 |
| 4 | **组件测试类** | 100% 组件覆盖、覆盖率 > 85%、交互/状态/样式/无障碍测试通过 |
| 5 | **单元框架类** | Vitest 配置完成、工具函数完善、CI/CD 集成、测试 < 5min |
| 6 | **闭环验证类** | 功能/质量/文档/部署/兼容性/性能全部验证通过 |
| 7 | **各种统一类** | 设计语言/代码规范/文档风格/交互体验完全统一 |
| 8 | **现状审核分析建议类** | 审核报告全面、问题准确、建议可行、计划清晰 |
| 9 | **MVP功能拓展类** | 拓展功能与目标一致、技术可行、计划合理、文档完整 |
| 10 | **高级功能完善类** | 满足需求、实现稳定、性能优秀、UX 流畅、文档完整 |
| 11 | **性能优化类** | 首屏 < 2s、切换 < 100ms、内存 < 500MB、CPU < 30%、60fps |
| 12 | **安全加固类** | 无高/中危漏洞、低危 < 5、安全评分 > 90、通过渗透测试 |

## 执行流程

**基础阶段（必选）**: 1→2→3→4→5→6
**进阶阶段（推荐）**: 7→8→9
**高级阶段（可选）**: 10→11→12
**最终验收**: 整体评分 ≥ 90

## 整体评分标准

- 代码质量评分: ≥ 90
- 功能完整性评分: ≥ 90
- 测试覆盖率评分: ≥ 80
- 性能评分: ≥ 90
- 安全性评分: ≥ 90
- 文档完整性评分: ≥ 95
- 兼容性评分: ≥ 90
